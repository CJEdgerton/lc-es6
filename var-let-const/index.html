var and hoisting Explaination

Both blocks below perform the same. When assigning value to a variable using var, the variable declaration is "hoisted" to the top of the function. It essentially is declared but not defined throughout the function.

Check the console to use the functions. 


function myFoo(bool) {
	if( bool ) {
		var foo = 'bar';
		console.log(foo); // outputs 'bar'
	} else {
		console.log(foo); // outputs undefined
		console.log(myVar); // outputs uncaught reference error: myVar is not defined.
	}
}



function myBar(bool) {
	var foo;
	if( bool ) {
		foo = 'bar';
		console.log(foo); // outputs 'bar'
	} else {
		console.log(foo); // outputs undefined
		console.log(myVar); // outputs uncaught reference error: myVar is not defined.
	}
}

let

Defining variables with let does not hoist like var does. Refering to the example below:


function myBaz(bool) {
	if( bool ) {
		let foo = 'bar'; // using 'let' does not hoist the declaration
		console.log(foo); // outputs 'bar'
	} else {
		console.log(foo); // outputs uncaught reference error: foo is not defined
		console.log(myVar); // outputs uncaught reference error: myVar is not defined.
	}
}

const

Declaration with const disables reassignment on a variable. It does not however mean that it is immutable.


const colors = ['White', 'Gray', 'Black'];

colors = ['Red', 'White', 'Blue']; // Throws...  Uncaught TypeError: Assignment to constant variable.

/* *** However, using const does not mean that colors is immutable... *** */

colors.push('Magenta'); // This works and colors becomes...
console.log('colors'); // Outputs ['White', 'Gray', 'Black', 'Magenta'];
